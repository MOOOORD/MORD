# 非对称竞技2D小游戏 — 设计文档

## 概述

2D 俯视角像素风非对称竞技游戏。玩家操控求生者，对抗 AI 控制的监管者。灵感来自黎明杀机 / 第五人格。

## 技术方案

- **平台**: HTML5 Canvas + ES6 模块化 JavaScript
- **渲染**: 纯 Canvas 2D 像素风格，requestAnimationFrame + 60fps 固定逻辑更新
- **后续扩展**: 核心逻辑模块化，游戏逻辑与渲染分离，方便之后移植到 Pygame 桌面版
- **运行**: 浏览器直接打开 `index.html`

## 游戏模式

### 模式一：逃生模式
- 地图上有 5 台发电机，修好 **2 台** 即可给大门通电
- 大门通电后靠近长按开门，穿过大门逃脱胜利
- 受伤机制：健康 → 受伤（移速略降）→ 倒地
- 倒地后被挂上钩子，共 3 次判定即淘汰
- 每次上挂可挣扎尝试脱钩（成功率约 12%，脱钩后有短暂保护时间）

### 模式二：分数模式
- 5 分钟时限，以累积分数为目标
- 得分：修好发电机 +1000，砸中监管者 +300，修机每秒 +20
- 扣分：被击中 -200，倒地 -500
- 时间到或被淘汰结算总分

## 地图系统

三种可选布局，混合作战：

| 类型 | 特点 |
|------|------|
| **房间走廊式** | 多个房间通过狭窄走廊连接，转角多，适合躲藏 |
| **开阔场地式** | 大片空地散布障碍物和板子，偏追逐走位 |
| **混合式** | 边缘建筑 + 中央开阔区，玩法最丰富 |

地图固定包含：5 台发电机、2 扇大门、6 个钩子、若干板子点。

## AI 监管者

状态机驱动，自动切换行为：

| 状态 | 行为 |
|------|------|
| 巡逻 | 在未修好的发电机之间移动巡视 |
| 警觉 | 听到修机声 / 看到痕迹后前往调查 |
| 追逐 | 发现玩家后持续追击，速度略快于玩家 |
| 搬运 | 击倒玩家后抱起到最近钩子 |
| 破坏 | 清除路上的板子 |

## 操作

| 按键 | 动作 |
|------|------|
| WASD / 方向键 | 移动 |
| 空格 | 交互（修机 / 开门 / 放板子 / 挣扎） |
| Shift | 奔跑（体力条限制） |
| Tab | 标注发电机/大门位置 |
| Esc | 暂停菜单 |

## 视觉风格

- 像素风 2D 俯视角
- 角色 2-4 帧像素动画
- 求生者：彩色小人，受伤带血滴粒子；监管者：暗色大体型，移动带残影
- 发电机未修好时冒火花闪烁
- 心跳指示：屏幕四角红色渐变 + 像素心跳图标

## UI 布局

- 左上：健康状态（❤️❤️ / 💔❤️ / 💀）
- 下方中央：交互进度条
- 右上：剩余发电机数 + 大门状态
- 底部：道具/技能栏（预留扩展）
